PROJECT KA SCENE
-----------------
Bhai basically ek full-stack web app hai jisme students apna caution money
wapas maang sakte hain after college khatam hone ke baad. Backend mein
Node.js, Express aur MongoDB use kiya — classic "jo sab karte hain wahi karo"
stack. Multi-role admin clearance workflow hai, matlab 5 departments khush
hone chahiye tabhi paisa milega. Bohot hi "sarkari" feel hai but digitally.

Tech Stack : Node.js, Express.js, MongoDB (Mongoose), JWT, BCrypt
Architecture: MVC + Repository + Service (fancy bolne ke liye, basically
              folders ke andar folders ke andar folders)

KYA KIYA:

    Literally "bhai ye karte hain" bolke project shuru kiya aur baaki sab
    ko kaam de diya. CEO energy.
  - System ka poora structure design kiya — MVC + Repository + Service pattern.
    Baaki log ek folder banate, isko pura galaxy-brained architecture banana tha.
  - Teen database schemas design kiye (yahi toh asli nerd work hai):

      * Student Schema (src/models/Student.js)
          - Enrollment number, department, bank details, DOB — sab plan kiya
          - Security fields bhi — loginAttempts, lockUntil
            (5 baar wrong password? 2 ghante ke liye ban. Strict hai yaar.)
          - Methods: comparePassword(), isLocked(), incLoginAttempts()

      * Admin Schema (src/models/Admin.js)
          - 6 types ke admin: superadmin, library, sports, hostel,
            department, accounts. Hierarchy hai yahan bhi, barabar hai.
          - Admins ko bhi same security treatment — no VIP exemption

      * Application Schema (src/models/Application.js)
          - 5 departments ki clearance ka pura system ek schema mein fit kar diya
          - Timeline array — audit trail, matlab koi bhi "maine nahi kiya"
            nahi bol sakta. Receipts hain sab ki.
          - Pre-save hook jo automatically overall status update karta hai.
            Schema khud sochta hai. Literally smarter than us.
          - Application ID generation (IEHE-[timestamp] format)
          - noDuesMode aur noDuesImageId fields add kiye application root pe —
            student ne offline form submit kiya ya nahi, aur agar kiya toh
            kaun si image. Har clearance pe alag nahi — ek form, ek image,
            poori application ke liye. Clean design.
          - Teen naye image ID fields add kiye — document upload Step 3 ke liye:
              tcOrAdmissionSlipImageId  → TC ya PG admission slip
              bankPassbookImageId       → Bank passbook front page
              feesSlipImageId           → Fees receipt
            Teeno ObjectId hain (ref: Image) — null jab tak student upload na kare.
            Schema change seedha — baki sab infrastructure already tha. Smart.

      * Image Schema (src/models/imageModel.js)
          - Naya schema banaya images store karne ke liye
          - fileName, mimeType, data (Buffer) — binary image seedha MongoDB mein
          - timestamps automatic — kab upload hua, sab record

  - server.js setup kiya — pura middleware stack, rate limiting, health check,
    centralized error handling. Foundation rakhi poori building ki.
  - Image save pipeline banaya — do utilities jo milke kaam karti hain:
      * saveImageToDb(buffer, originalName, mimeType) — compress karo, MongoDB
        mein save karo, imageId return karo. Disk ka koi scene nahi — pure
        in-memory. Vercel pe bhi kaam karta hai.
      * imgCompress(buffer, mimeType, opts) — sharp se compress karta hai.
        Agar fail ho toh original buffer return — graceful fallback built-in.
  - Saare constants define kiye:
      * departments.js  — DEPARTMENTS, CLEARANCE_TYPES, CLEARANCE_ROLE_MAP
      * statuses.js     — APPLICATION_STATUS, CLEARANCE_STATUS, REFUND_STATUS
      * roles.js        — ADMIN_ROLES, USER_TYPES
    (basically poori duniya ki vocabulary project ke liye define ki)
  - src/ ka modular folder structure establish kiya — baaki sab ko
    andhere mein roshni di. Unsung hero moment.

6. AYUSH  |  "Koi hack nahi karega mere rehte" guy  |  Security Developer
--------------------------------------------------------------------------------
ROLE: Backend ka Bouncer. Security Infrastructure ka Boss.

KYA KIYA:
  - Core auth middleware banaya (src/middleware/auth.js):
      * protect()        — JWT verify karo, req.user populate karo. No token? Bye.
      * protectStudent() — Sirf student. Admin ne try kiya? Nahi chalega.
      * protectAdmin()   — Sirf admin. Student ne try kiya? Bhai wrong floor.
      * restrictTo()     — Specific role? Specific kaam. Koi jugaad nahi.
  - Helmet.js lagaya — security headers automatic. Hacker dekhe toh bhage.
  - CORS configure kiya — sirf frontend URL se baat karega. Stranger danger real hai.
  - Two-tier rate limiting set kiya:
      * General API: 100 requests/15 min per IP — normal usage ke liye theek hai
      * Auth endpoints: sirf 10/15 min — brute force karne ki soch bhi mat
  - Request body 10KB se zyada? Hard reject. Server buffet nahi hai bhai.
  - Morgan HTTP logging — sab record ho raha hai. Behave karo.
  - env.js — startup pe required env vars check. Koi missing? Process exit.
    No env, no party. Zero tolerance.
  - db.js — MongoDB connection. Simple but iske bina sab kuch zero hai.
  - multerMiddleware.js banaya — student image upload ke liye:
      * memoryStorage use kiya — disk pe kuch nahi likhta. Vercel ka filesystem
        read-only hai, toh diskStorage wahan fail hoti. Memory mein rakho,
        seedha MongoDB bhejo. No temp files, no ENOENT, no drama.
      * Sirf images accept karta hai (image/* MIME types). 5MB limit.
      * req.file.buffer mein milta hai file — clean aur simple.
  - server.js mein image routes mount kiye — /api/image pe. Bina iske image
    retrieve hoti hi nahi. Foundation ka kaam aise hi hota hai.

KEY FILES:
  src/middleware/auth.js | src/middleware/multerMiddleware.js
